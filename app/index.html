Project documentation

Overview

An end-to-end DevSecOps pipeline built to demonstrate how modern security practices integrate natively into a software delivery workflow — not bolted on at the end. Every code push triggers an automated chain: vulnerability scans run before any image ships, deployment is driven by Git state rather than manual commands, and every container action in the cluster is watched for suspicious behaviour.

Pipeline flow
Developer (git push) │ ▼ GitHub Repository source code + Kubernetes manifests │ ▼ GitHub Actions CI/CD ├── Docker build ARM64 + AMD64 multi-arch ├── Trivy scan block on HIGH / CRITICAL CVEs ├── Push to Docker Hub versioned image tag └── Update deployment.yaml manifest auto-update (in progress) │ ▼ Argo CD watches Git → deploys automatically │ ▼ Kubernetes (Minikube) │ ┌───────────┼──────────────┐ │ │ │ ▼ ▼ ▼ Resume app Falco Prometheus │ │ ▼ ▼ Falcosidekick Grafana
Component breakdown
ComponentWhat it does
DockerPackages the resume website into a container. Runs as a non-root user with minimal Linux capabilities — if the container is compromised, the attacker has very limited reach.
GitHub ActionsAutomates the full build-scan-push-deploy chain on every commit. No manual steps between code and cluster.
TrivyScans both the filesystem and the final Docker image for known CVEs. Any High or Critical finding blocks the pipeline — nothing ships until it's clean.
Docker HubStores versioned image tags. Only images that passed the Trivy gate land here.
Argo CDWatches the Git repository for manifest changes. When a new image tag appears, it automatically synchronises the Kubernetes cluster — the Git repo is the single source of truth for what runs.
KubernetesRuns the application on a Minikube cluster with NGINX Ingress for traffic routing. Pod Security Contexts enforce non-root execution and drop default capabilities.
FalcoContainer security agent running inside the cluster. Watches every container action at the kernel level — shells spawned, API server contacts, unexpected file writes — and fires a structured alert when a rule matches.
FalcosidekickRoutes Falco alerts to Prometheus for metric aggregation and persistence.
PrometheusScrapes Falco metrics on a configured interval, stores them as time-series data for querying.
GrafanaVisualises Prometheus data across panels: total alerts, severity breakdown, top rules, alerts by pod, and a live event feed.
Security principles
PrincipleImplementation
Shift leftTrivy scans happen before the image reaches any registry — vulnerability gates are in the pipeline, not an afterthought.
Immutable infraNo one logs into a server to make changes. Every change goes through Git and is applied by Argo CD.
Least privilegeContainers run non-root with dropped capabilities. Minimal blast radius on any breakout.
Runtime visibilityFalco provides continuous monitoring after deployment — unexpected container behaviour is detected and recorded immediately.
GitOpsGit is the source of truth. Cluster state is always a direct reflection of the repository — no drift, no snowflakes.
